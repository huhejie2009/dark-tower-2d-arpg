# 2026-06-04 新项目首个可玩骨架进度

## 项目位置

新项目已建立在：

`H:\GODOT_PROJECT\dark-tower-2d-arpg`

入口文件：

`H:\GODOT_PROJECT\dark-tower-2d-arpg\project.godot`

本轮没有复制旧 3D 主线、POLYGON 资源或旧项目里的不稳定场景。

## 本轮目标

旧项目已经出现依赖缺失、卡死、无法移动、无法攻击等阻断问题，因此本轮改为在 H 盘建立一个干净的 Godot 4.6.2 纯 2D 项目，并优先做到“能打开、能走、能打、能清怪进下一层”的首个可玩骨架。

## 已完成内容

- 新建 Godot 项目配置，主入口为 `res://scenes/MainMenu.tscn`。
- 新建主菜单、职业选择、主城、2D 战斗四个核心场景。
- 新建 2D 玩家、敌人、掉落物、HUD、基础技能和轻量特效脚本。
- 新建纯数据服务：玩家数据、背包、装备、爬塔进度、存档结构。
- 新建职业规则、技能规则、装备词条规则、掉落规则。
- 建立基础存档流程：创建角色、读取当前角色、保存当前角色、清层奖励记录。
- 建立第一版战斗循环：WASD/方向键移动、左键基础攻击、敌人追击、敌人死亡掉落、拾取进背包、清怪开传送门、按 E 或进入传送门前往下一层。
- 修复新项目启动阻断风险：Godot 脚本类型推断警告、脚本类名加载顺序、传送门信号连接写法。
- 修复“左键像没攻击”的早期手感风险：近身敌人存在时，左键会优先校正到最近可攻击目标方向。
- 修复清层后保存快照问题：进入下一层、回城、死亡时保存当前玩家生命、背包、楼层信息。

## 当前可玩状态

当前版本不是最终游戏，但已经具备最小可玩闭环：

1. 打开主菜单。
2. 进入职业选择。
3. 创建角色进入主城。
4. 从主城进入爬塔战斗。
5. 移动、攻击、击杀敌人。
6. 拾取掉落。
7. 清怪后开传送门。
8. 进入下一层或回城。
