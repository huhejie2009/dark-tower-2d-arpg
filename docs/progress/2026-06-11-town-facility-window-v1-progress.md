# 2026-06-11 主城设施窗口 V1 进度

## 本轮目标

把主城中的商人、铁匠、仓库、训练从“直接复用背包窗口”的临时状态，推进为独立设施入口。当前先完成可交互窗口骨架和动作桥接，不制作代码生成素材，不重做物品数据。

## 已完成内容

- 新增 `TownFacilityService`，集中维护主城设施配置。
- 新增 `TownFacilityWindow`，用于展示设施标题、说明和动作按钮。
- 商人、铁匠、仓库、训练交互点改为先打开设施窗口。
- 设施动作再桥接到现有背包/装备窗口：
  - 商人：打开背包、卖出废品。
  - 铁匠：查看装备、分解废品。
  - 仓库：预留独立仓库系统，当前可先查看背包。
  - 训练：打开技能页。
- 主城移动逻辑会在设施窗口打开时暂停角色移动，避免窗口打开后误走位。
- 新增商人设施截图 QA 工具与截图。

## 验收标准

- 四个主城设施都有稳定配置、标题、描述和动作列表。
- 主城交互不会立即强行打开背包窗口。
- 设施窗口在 1280x720 下不出屏、不遮挡右侧整备栏。
- 设施动作可以继续复用现有背包筛选、卖废品、分解废品和技能页逻辑。
- 后续正式商人、铁匠、仓库系统可以挂接在 `TownFacilityService` 和 `TownFacilityWindow` 之后，不需要再改主城交互点。

## 后续建议

1. 做仓库数据服务：独立容量、存取物品、不占背包容量。
2. 做商人买回列表：卖出废品后保留短期回购池。
3. 做铁匠分解确认和材料仓库：与现有废品分解服务共用预览逻辑。
