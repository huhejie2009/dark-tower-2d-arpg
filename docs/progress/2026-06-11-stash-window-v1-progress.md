# 2026-06-11 仓库窗口 V1 进度

## 本轮目标

在已有 `StashStorageService` 数据底座上，接入一个可实际使用的主城仓库窗口。当前优先保证存取逻辑、窗口层级和接口稳定，不制作代码生成素材，不做最终美术。

## 已完成内容

- 新增 `StashWindow`。
- 主城仓库设施的 `Open Stash` 动作会打开独立仓库窗口。
- 仓库窗口采用三栏结构：
  - 左侧：背包物品列表与背包容量。
  - 中间：仓库物品列表与仓库容量。
  - 右侧：操作说明与结果提示。
- 支持整件/整栈存入仓库。
- 支持整件/整栈从仓库取回背包。
- 打开仓库时会关闭设施面板，避免窗口互相遮挡。
- 仓库操作后保存玩家背包数据和当前存档槽仓库数据。
- 新增仓库窗口截图 QA 工具与截图。

## 验收标准

- 主城仓库设施可以打开独立 `StashWindow`。
- 打开仓库不会强制打开背包/装备窗口。
- 仓库窗口在 1280x720 下完整位于屏幕内。
- 存入后背包列表移除该物品，仓库列表出现该物品。
- 取回后背包列表出现该物品，仓库列表移除该物品。
- 打开仓库时设施面板自动收起，避免层级遮挡。

## 后续建议

1. 仓库窗口接入搜索、筛选、排序。
2. 用格子化图标列表替换当前文字按钮列表。

4. 后续正式素材进入后，只替换图标和面板资源，不改存取数据逻辑。
