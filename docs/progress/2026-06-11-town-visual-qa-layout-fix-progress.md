# 2026-06-11 主城视觉 QA 与布局修复进度

## 本轮目标

针对试玩反馈中“上次生成的主城有问题”的部分，先用截图确认问题，再修复主城可玩空间与 UI 面板的遮挡关系。

## 截图确认的问题

- 主城整备面板仍然沿用菜单页居中布局，覆盖了主城世界、玩家和通天塔入口。
- `Main Menu` 按钮在 1280x720 下贴近底边，存在出屏风险。
- 互动提示位于视口下方，玩家难以获得“可移动 / 可交互”的反馈。
- 主城世界虽然已经存在，但因为 UI 层级和空间分区错误，实机观感更像菜单页而不是可走动主城。

## 已完成修复

- 将主城改为左侧可行走世界、右侧整备栏的屏幕分区。
- 将通天塔入口、玩家、商人、铁匠、仓库、训练点全部移入左侧可见主城空间。
- 收窄右侧 `TownPrepPanel`，避免侵占可玩区域。
- 调整主城按钮组到右侧下方，确保 720p 下全部可见。
- 调整玩家移动边界，避免角色走进右侧 UI 面板或贴到屏幕底部。
- 新增主城截图 QA 工具：`tools/qa_capture_town_screenshot.gd`。
- 新增视觉布局回归：`regression_town_playable_space_visual_layout.gd`。

## 验收标准

- 1280x720 下整备面板位于右侧，不遮挡玩家和塔门。
- 玩家初始位置在主城可玩区域内。
- 通天塔入口与交互标记清晰可见。
- 互动提示可见。
- 主城所有按钮完全位于视口内。
- 主城既保留现有背包 / 装备 / 技能入口，又为后续商人、铁匠、仓库独立窗口预留接口。
