## 已确认并修复的问题

### 1. 背包窗口在小视口下越界

问题：

- `InventoryEquipmentWindow` 原本固定为约 860x500。
- 在 800x600 或更小窗口中，窗口会超出屏幕右侧和底部，影响玩家查看装备、技能和操作按钮。

修复：

- 背包窗口保留居中显示，但会根据当前视口尺寸自动收缩，并保留边距。
- 新增响应式布局计算接口，方便后续不同分辨率测试复用。

新增验证：

- `NEW_PROJECT_INVENTORY_WINDOW_RESPONSIVE_BOUNDS_OK`

### 2. 新增脚本的 Godot UID 文件漏入库

问题：

- Godot 为新增脚本生成了 `.gd.uid` 文件。
- 项目中既有 `.gd.uid` 已纳入版本库，新增 UID 不提交会导致工作区反复变脏，也可能影响协作机器上的资源识别稳定性。

修复：

- 将本轮发现的新增 `.gd.uid` 文件纳入提交范围。

### 3. README 最近验证计数过期

问题：

- README 中仍记录 `ALL_NEW_PROJECT_REGRESSION_OK COUNT 100`。
- 当前回归数量已经增加，旧计数会误导后续接手者。

修复：

- 更新 README 验证标记为当前回归数量。
- 进程残留描述改为更精确的 `NO_RESIDUAL_HEADLESS_TEST_GODOT_PROCESS`。
