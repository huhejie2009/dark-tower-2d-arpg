# 2026-06-11 商人交易数据服务进度

## 本轮目标

根据最新反馈，暂时停止在当前占位 UI 上继续美化，转向功能底座。本轮实现商人交易数据服务，为后续正式商店窗口、买回列表和批量出售提供稳定规则。

## 已完成内容

- 新增 `VendorTransactionService`。
- 支持卖出单个背包物品。
- 卖出后会把物品原始数据写入买回池。
- 支持从买回池买回物品。
- 锁定、收藏、不可出售、已装备物品不会被卖出。
- 买回池默认最多保留 12 条。
- V1 买回价等于卖出价，优先保证“误卖可恢复”；后续经济系统再统一调数值。

## 新增回归

- `regression_vendor_transaction_rules.gd`

## 验收标准

- 未保护、未装备物品可卖出并获得金币。
- 卖出后物品从背包移除，进入买回池。
- 锁定/收藏/已装备物品卖出失败，并返回 `protected_item`。
- 买回成功后物品回到背包，买回池移除对应条目。
- 金币不足时买回失败，并返回 `not_enough_gold`。
