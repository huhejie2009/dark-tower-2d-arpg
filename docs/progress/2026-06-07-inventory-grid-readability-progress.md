# 2026-06-07 背包格子可读性增强进度

## 本轮目标

继续提升背包、装备等游戏功能，先让刷宝结果在背包中更容易判断。

## 已完成

- 背包格子加入稀有度视觉信息：
  - 普通、魔法、稀有、传奇、材料、货币分别有稳定颜色接口。
  - 当前先用按钮底色和边框表现，后续可直接替换为正式图标格子皮肤。
- 背包格子加入状态标识：
  - `E`：当前已装备。
  - `+`：同槽位中比当前装备分数更高。
- 新增物品视觉元数据接口：
  - `get_item_visual_metadata_for_test(item_id)`
  - 返回稀有度、格子标签、徽标、已装备状态、升级候选状态、边框色、背景色。
- 装备分数接口继续复用上一轮的 `Score` 逻辑，后续可迁移到独立装备评分服务。

## 回归覆盖

新增：

- `tests/regression/regression_inventory_item_visual_metadata.gd`

覆盖内容：

- 已装备物品会标记 `E`。
- 同槽位更强装备会标记 `+`。
- 视觉元数据包含稀有度、颜色和格子标签。

- 加战斗内掉落提示：更强装备、新稀有装备、Boss 奖励单独提示。
- 把装备评分迁移到 `EquipmentDataService`，供掉落提示、自动排序和装备推荐共用。
