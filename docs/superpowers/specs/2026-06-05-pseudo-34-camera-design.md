# 2026-06-05 伪 3/4 俯视视角设计

## 方向确认

本轮视角方向采用 C 方案：伪 3/4 俯视。

目标不是复制《元气骑士》的素材或具体画面，而是让当前 2D 暗黑刷宝/爬塔 ARPG 获得更强的房间立体感、角色存在感和弹幕可读性。

## 设计目标

- 战斗画面从纯平面大矩形竞技场，转向“略带斜视感的房间式俯视”。
- 玩家、敌人、掉落、投射物仍使用 Godot 2D 逻辑坐标，不把玩法系统改成真实斜坐标。
- 伪 3/4 只作用于表现层：地面、墙体、障碍、角色外观比例、阴影和视觉层级。
- 保持现有移动、攻击、敌人追击、掉落拾取、传送门、楼层奖励和存档逻辑稳定。

## 视觉规则

### 镜头

- Camera2D 继续挂在玩家身上。
- 镜头更接近当前房间，让玩家和敌人的屏幕占比提高。
- 不做强烈旋转镜头，避免鼠标瞄准、投射物方向和碰撞边界产生认知偏差。

### 房间

- 当前 `room_rect` 仍作为真实逻辑边界。
- 视觉上将房间绘制成轻微斜切的地板轮廓。
- 地砖线可做水平/竖向压缩，让地面看起来有透视感。
- 房间边缘用更厚的墙边和暗色内描边强调封闭感。

### 单位

- 玩家和敌人不再只像纯圆点，而是采用“椭圆身体 + 底部阴影 + 朝向提示”的程序化表现。
- 身体高度略大于宽度，形成轻微立体角色感。
- 阴影保持在逻辑脚底位置，帮助玩家判断实际碰撞中心。

### 投射物和技能警示

- 投射物仍沿真实 2D 方向飞行。
- 视觉上可以加短拖尾和亮边，不倾斜命中逻辑。
- Boss 警示区、死亡爆炸圈等仍按真实圆形/范围判定显示，后续可加透视压缩版本，但第一版不改判定。

## 技术边界

第一版只改表现，不改核心系统：

- 不改存档 schema。
- 不清除玩家存档。
- 不改背包、装备、楼层奖励数据。
- 不恢复旧 3D 项目资源。
- 不恢复 POLYGON 资源。
- 不引入 3D 节点或 3D 摄像机。

## 推荐实现步骤

1. 新增视觉规则脚本或常量，用于统一镜头缩放、单位绘制比例、地面颜色和墙体颜色。
2. 调整 `Game2D.gd` 中的 Camera2D 缩放和房间绘制。
3. 把 `DungeonFloor` 从普通矩形表现升级为伪 3/4 地板轮廓和压缩地砖线。
4. 调整 `Player2D.gd` 与 `Enemy2D.gd` 的程序化外观比例和阴影。
5. 保持 `room_rect`、移动 clamp、敌人追击和技能判定不变。
6. 增加回归测试，确认视角配置、房间边界和核心玩法入口仍稳定。

## 测试策略

- 新增视角契约测试，确认 Game2D 暴露或创建了伪 3/4 视觉节点。
- 保留现有场景启动烟测。
- 保留完整回归，重点确认：
  - 玩家仍可移动。
  - 敌人仍可生成、追击、死亡。
  - 掉落仍可拾取。
  - 清怪仍会开传送门。
  - 死亡结算和暂停 UI 不受影响。

## 风险与处理

- 风险：伪 3/4 地板看起来斜，但碰撞仍是矩形，玩家可能误判墙边。
  - 处理：第一版让真实边界线和阴影清楚可见，不做过度斜切。
- 风险：角色变高后，实际碰撞中心不清楚。
  - 处理：增加脚底阴影，碰撞感以阴影为准。
- 风险：投射物和范围警示被视觉透视影响可读性。
  - 处理：第一版保持投射物和范围提示按真实平面显示，优先保证战斗可读。

## 验收标准

- 打开战斗场景后，整体画面明显不再像纯平面矩形竞技场。
- 玩家、敌人和房间具备轻微 3/4 俯视观感。
- 玩家移动、攻击、清怪、拾取、传送门、暂停、死亡结算功能不回退。
- 完整回归通过。
