# 2026-06-09 UI 参考边界说明

## 核心结论

游戏内 UI、界面布局、信息层级、背包/装备可读性，可以参考《暗黑破坏神 4》这类现代暗黑 ARPG 的成熟做法。

但这不是美术风格迁移。项目整体美术仍然维持自己的方向：

- 冷峻水泥质感的通天塔巨构。
- 草原、平原、河流与远景世界叙事。
- 冰冷、荒凉、神秘，但不是直接暗黑破坏神式恶魔哥特。
- 角色、怪物、环境、概念图、过场图继续遵守 Dark Tower 自己的美术设定。

## 允许参考的部分

- HUD 信息结构：生命、魔力、经验、日志、掉落提示要清楚，不遮挡战斗。
- 背包与装备窗口：物品格、稀有度、筛选、装备对比、推荐提示要一眼能读。
- 界面交互状态：按钮悬停、按下、选中、禁用必须明显。
- 暗黑 ARPG 的读屏节奏：先看生存资源，再看战斗反馈，再看掉落收益。

## 不允许参考的部分

- 不把环境、角色、怪物、塔楼、概念图改成《暗黑破坏神 4》的美术方向。
- 不复制暗黑 4 的图标、纹样、字体、装饰件、UI 素材。
- 不把现有冷峻巨构塔风格改成恶魔哥特、地狱祭坛、火焰炼狱或圣堂风。
- 不因为 UI 使用暗红、暗金、骨白，就改变游戏世界观和素材生产方向。

## 当前落地方式

`scripts/ui/DarkArpgUiTheme.gd` 是项目自己的 UI 主题层，主题 ID 为：

`dark_tower_brutalist_arpg_ui_v1`

这个名字刻意不包含外部游戏名，表示它属于 Dark Tower 项目自己的 UI 体系。暗黑 4 只作为 UI/UX 参考来源，不作为美术资产或世界观参考来源。

## 后续验收标准

- 新 UI 必须接入统一主题层，而不是每个界面各写一套颜色。
- 新素材接口必须继续服务 Dark Tower 自己的巨构塔美术方向。
- 回归测试应防止主题 ID 重新出现外部游戏名，避免团队误解。
- 视觉 QA 时分别检查两件事：界面是否更清晰，游戏美术是否仍保持自有风格。
