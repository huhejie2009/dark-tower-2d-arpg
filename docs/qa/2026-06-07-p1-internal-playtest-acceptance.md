# P1 内部试玩验收清单

日期：2026-06-07  
阶段：P1 可试玩 UI 与装备闭环  
目标时长：至少 30 分钟  
素材原则：本轮验收不新增代码生成素材，重点检查系统闭环、UI 可读性、存档安全和可扩展接口。

## 通过规则

- P0 阻塞缺陷必须为 0。
- P1 严重缺陷不超过 2 个。
- 完整回归必须通过，并输出 `ALL_NEW_PROJECT_REGRESSION_OK`。
- 主项目 headless 启动退出码必须为 0。
- 不允许清除玩家存档。

## 缺陷分级

- P0：阻塞继续游玩、丢存档、无法启动、死亡/回城后角色无法继续。
- P1：核心功能可用但误导玩家，例如误穿装备、背包状态不同步、死亡结算信息错误。
- P2：体验问题，例如提示停留时间不理想、文案不够清楚、布局需要 polish。
- P3：低优先级视觉或文案调整。

## 人工试玩路线

1. 从主菜单进入角色/存档流程。
2. 进入主城，打开背包/装备窗口，检查纸娃娃面板和装备槽。
3. 进入战斗，移动、攻击、拾取 3 到 5 个掉落。
4. 打开背包，比较装备，装备一件更强装备，再卸下一次。
5. 清怪，确认传送门出现，进入下一层。
6. 重复至少 3 层，期间测试暂停、打开背包、回城。
7. 主动死亡一次，检查死亡结算、回城半血、背包和装备保留。
8. 退出并重新启动，确认角色数据仍在。

## 进入 P2 的条件

- P1 验收通过。
- P0 为 0。
- P1 不超过 2 个，并已记录到后续修复清单。
- 完整回归、场景烟测、主项目启动验证均通过。

## P2 前建议处理

- 掉落提示停留时间和样式可以继续 polish，但不阻塞进入 P2。
- 装备槽稀有边框和图标接口可以与 P2 战斗手感并行推进。
- 不要在 P2 前新增代码生成素材；素材替换继续走 IMAGE2/人工资产管线。
